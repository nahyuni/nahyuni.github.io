Pipe 기반 소켓 처리

개요

대용량 스트리밍 처리, 빠른 응답성 요구, 메시지 기반 통신
에 적합하다.

구조

이 패턴은 Producer-Consumer 구조로, 소켓 수신자는 Pipe에 데이터만 적재(Write) 데이터 처리는 Pipe에서 소비(Read)

Pipe 기반의 처리 장점:

버퍼 풀링: GetMemory()로 기존 메모리를 재사용
Producer-Consumer 분리: 읽기/쓰기 병렬 가능
Zero-Copy 가능성: 성능 극대화
